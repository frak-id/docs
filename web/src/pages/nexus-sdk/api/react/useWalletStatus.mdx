# useWalletStatus

## Summary
The `useWalletStatus` hook provides real-time updates on the current status of the Nexus Wallet. It allows React components to react to changes in wallet connection, user authentication, and interaction session status.

## Requirements
- Must be used within a component wrapped by `NexusIFrameClientProvider`
- Requires an initialized Nexus client

## Hook Signature

```ts twoslash
// @noErrors
import { UseQueryResult } from '@tanstack/react-query';
import type { WalletStatusReturnType, FrakRpcError } from '@frak-labs/nexus-sdk/core';

declare function useWalletStatus(): UseQueryResult<WalletStatusReturnType, FrakRpcError>;
```

### Return Value
- `UseQueryResult<WalletStatusReturnType, Error>`: A query result object from React Query

## WalletStatusReturnType

The `WalletStatusReturnType` is defined as:

```ts twoslash
// @noErrors
import { Address } from 'viem';

type WalletStatusReturnType = Readonly<WalletConnected | WalletNotConnected>;

type WalletConnected = {
  key: "connected";
  wallet: Address;
  interactionSession?: {
    startTimestamp: number;
    endTimestamp: number;
  };
};

type WalletNotConnected = {
  key: "not-connected";
};
```

### Properties
- When connected:
    - `key`: `"connected"`
    - `wallet`: `Address` - The Ethereum address of the connected wallet
    - `interactionSession` (optional): Object containing session timestamps
        - `startTimestamp`: `number` - Start time of the current interaction session
        - `endTimestamp`: `number` - End time of the current interaction session

- When not connected:
    - `key`: `"not-connected"`

## Notes
- This hook uses React Query's real-time query functionality, providing automatic updates when the wallet status changes.
- It internally sets up a listener using the `watchWalletStatus` action.
- The hook handles connection, disconnection, and interaction session updates automatically.
